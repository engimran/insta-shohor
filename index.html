Q:Difference of Local Storage and Session Storage

LocalStorage and SessionStorage are web storage objects that allow key / value pairs to be stored in the browser. The funny thing about them is that the data survives even after a page refresh (for sessionStorage) and even a complete browser restart (for localStorage). The sessionStorage object is much less commonly used than the localStorage object. Although the session storage properties and methods are the same, they are very limited. • sessionStorage only exists in the Browser Current tab. Another tab on the same page will have separate storage. • Data pages survive after refresh, but not when tabs are closed or new tabs are opened.
